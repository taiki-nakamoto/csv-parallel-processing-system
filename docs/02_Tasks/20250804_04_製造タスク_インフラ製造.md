## 2. インフラ製造概要

### 2.1 インフラ製造の位置付け
製造準備フェーズ完了後の第二段階として、以下のインフラ製造を実施します：

1. **Terraform基盤構築**: プロバイダー設定、Backend設定、共通モジュール作成
2. **ネットワークインフラ構築**: VPC、サブネット、セキュリティグループ
3. **データベースインフラ構築**: Aurora PostgreSQL、DynamoDB
4. **ストレージインフラ構築**: S3バケット、ライフサイクルポリシー
5. **監視・ログインフラ構築**: CloudWatch、X-Ray
6. **IAMロール・ポリシー構築**: Step Functions用、Lambda用ロール

## 4. インフラ製造実行順序

### 4.1 Phase 1: 基盤構築（推定時間: 3時間）
1. **Terraformプロジェクト初期化** (30分)
   - プロバイダー設定
   - Backend設定
   - Variables定義

2. **共通モジュール作成** (120分)
   - ネットワークモジュール
   - IAMモジュール
   - S3モジュール
   - DynamoDBモジュール
   - CloudWatchモジュール

3. **初期検証** (30分)
   - terraform init
   - terraform validate
   - terraform plan

### 4.2 Phase 2: ネットワーク・セキュリティ（推定時間: 2時間）
1. **VPC・サブネット構築** (60分)
2. **セキュリティグループ構築** (30分)
3. **ルートテーブル・ゲートウェイ** (30分)

### 4.3 Phase 3: データベース・ストレージ（推定時間: 2時間）
1. **Aurora PostgreSQL構築** (60分)
